# yanxu-package 1.1.18

> 1.1.18 未形成稳定共享包 Release；以下内容最终由 1.1.20 正式交付。请直接
> 使用`yanxu-package` 1.1.20。

原定版本保持清单格式 2、锁文件格式 2 与公开 Rust API 的补丁兼容边界。

- 路径、Git 和注册表依赖通过不可变 generation 与已打开目录能力绑定校验结果，缓存损坏时保留旧现场并增量修复。
- 清单、锁文件、辖制清单、注册表索引、模块源码和原生制品使用有界普通文件快照，拒绝链接、重解析点、特殊文件和身份漂移。
- 打包阶段将入口、资源、图标和原生制品绑定到已打开包根及稳定内容快照，校验后的路径替换或同长原地改写不能改变归档字节。
- 归档校验同时约束压缩输入与完整解码预算，并拒绝附加 gzip member、解码尾随数据、链接、特殊文件和可移植路径碰撞。
- 源码应用、YXB 和工作区成员共用保留目录与可移植路径规则；应用资源及成员清单保持绑定发现阶段的已打开根，不跟随替换树或嵌套包越界。
- Git 提交检查、初始化、归档和抓取分别受硬截止时间、输出上限、磁盘预算与完整进程树回收约束。
- 注册表旧目录只作为复验迁移输入；`vendor`通过同父目录事务、持久化同步和状态化回滚保护已有内容。
- 依赖来源校验拒绝凭据、敏感查询、片段、控制字符和参数注入，审计覆盖精确修订、许可证、漏洞、撤回状态、目标与原生制品属性。
- 共享包随正式制品提供许可证正文、确定性 CycloneDX 1.5 SBOM、构建元数据、SHA-256 与构建来源证明。
- Rust 最低支持版本为 1.89；既有格式 1/2 清单和锁文件无需迁移。
