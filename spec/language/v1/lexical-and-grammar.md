# 词法与文法

源文件必须是 UTF-8。标识符可以使用 Unicode 字母、数字与下划线，但不得以数字起始，也不得与关键词相同。关键字、内建类型和值名区分大小写。换行通常等同空白；`#`和`//`引出行注释，`#:`和`///`紧邻公开声明时也是文档注释。

实现必须接受全角`（），：；【】`；当前兼容输入也接受对应半角括号、逗号、冒号和分号。文字可用`「」`、`“”`或`""`，支持`\n`、`\t`、`\r`、`\\`和引号转义。数字为十进制有限浮点值。

顶层由声明和语句组成。绑定为`令/定 名[:类型] 为 表达式；`，改写为`置 目标 为 表达式；`。块由`则`开始、`终`结束。条件、循环和遍历分别使用`若/否则`、`当`、`逐…于…`。函数为`[公] [异] 法`，类为`[公] 类`，协议为`[公] 协`，模块导入为`[公] 引 文字 为 名；`。`公 引`只能出现在顶层并公开导出模块命名空间；普通`引`仍是当前模块的私有依赖。`父`和`是`为保留关键词。

类型名是一个或多个以`.`连接的标识符段。单段`视图`表示当前模块类型，`基础.视图`表示导入模块类型，`界面.控件.按钮`表示经过公开重导出的任意深度路径。点号两侧不插入空格。限定路径可以递归出现在联合、可空、泛型和法类型中，也用于`承`、`纳`与`是`。

表达式优先级从低到高为：`或`、`且`、相等、类型判断`是`、比较、加减、乘除、一元`非/负号/候`、调用/成员/下标/切片、原子。逻辑与和或必须短路。`父.方法`是只可调用的绑定法表达式。切片使用半开区间；缺省起点为零，缺省终点为长度。

完整、可生成的编辑器文法以`tree-sitter-yanxu/grammar.js`为伴随文法；核心解析器是错误诊断和接受语法的权威实现，二者由同一语料持续校验。
