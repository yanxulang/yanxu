# 执行器与兼容性

参考树解释器与字节码 VM 必须在公开语言能力上产生相同的输出顺序、最终值、任务状态、模块初始化和错误类别。跨模块类身份、继承、协议、动态派发、`父.方法`、`是`与公开重导出也属于这项一致性要求。实现可以使用不同内部表示、缓存和垃圾回收策略，但不得静默回退至另一执行器。

每次宿主执行具有独立的步数预算；法调用深度与单一集合元素数也有上限。越界是可捕获前的普通运行失败，诊断必须指出`max_steps`、`max_call_depth`或`max_collection_elements`配置名。默认值属于实现安全基线而非源代码语义；宿主可以显式调整。

嵌入模式默认不具备文件、网络外连、TCP 监听、UDP 绑定、环境或进程能力。直接 CLI 为旧脚本兼容采用显式不受限模式；`包 运行`采用清单权限。静态模块摘要与运行时模块读取必须通过同一文件权限判定。网络外连在每次 HTTP 重定向和 DNS 解析后重新判定；回环、私网、链路本地及 CGNAT 地址接受精确 IP，或应用与宿主权限交集中的显式“本地网络”能力。其他特殊 IP 只接受精确授权，`*`不隐式扩大任一敏感地址范围。

持久嵌入引擎在运行阶段失败后必须保持运行时状态与静态类型历史一致。当实现无法事务回滚时，必须丢弃失败调用可能污染的全部持久状态；后续调用从空引擎继续。

`compat/`是 0.3 至当前版本的可执行兼容语料。`yanxu 兼容 --json`生成格式 1 报告。1.x 的补丁和次版本必须继续通过已有语料；新增语料可以收紧未规定的错误接受范围，但不得改变已规定的成功程序结果。
