# 公共格式

公共格式均以独立整数主版本标识。实现必须拒绝未知主版本，不得猜测解析。

| 格式 | 当前版本 | 标识 |
| --- | ---: | --- |
| 包清单 | 1 | `[包].格式`，缺省 1 仅为 0.7 迁移兼容 |
| 锁文件 | 1 | `lock_version` |
| 字节码块 | 1 | `Chunk.format_version` |
| 测试报告 | 1 | `schema_version`和`test-report-v1` URI |
| 兼容报告 | 1 | `schema_version`和`compatibility-report-v1` URI |
| C ABI 结果 | 1 | JSON `schema`字段 |
| 标准库清单 | 1 | `schema_version`和`stdlib-api-v1` URI |

格式 1 中可以增加接收方明确忽略的可选字段；删除字段、改变字段类型或语义、改变默认值都需要新主版本。锁文件和字节码要求精确主版本匹配；报告消费者应按 schema URI 选择解析器。

`yanxu::package`是包清单与锁文件格式 1 的规范解析实现。独立包管理器`yanbao`负责清单创作和依赖工作流，但必须复用核心解析器，不能另行定义格式语义。清单兼容言序惯用的中文裸键和标准 TOML 引号键；工具写回后两种形式都必须可读。

远程索引`tar.gz`制品只可包含相对路径的普通文件和目录。解压器必须拒绝父目录/绝对路径、符号链接、硬链接、块/字符设备、管道和重复文件写入，并在写入前检查压缩体积、条目数、路径长度、单文件和展开总量上限。安全拒绝不属于包格式破坏性变更。

路径依赖的内容校验按相对路径排序，将每个相对路径、文件字节长度和文件字节写入 SHA-256。`.git`、`.yanxu`、`target`目录、`.DS_Store`和任意层级的`言序.lock`属于工作区或派生制品，不参与内容校验；其他文件变化必须使旧锁失效。此排除集合属于格式 1 的可观察校验语义，工具不得因本机安装目录或操作系统元数据生成不同锁定结果。
