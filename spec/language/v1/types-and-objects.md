# 类型、对象与协议

内建类型为`数`、`文`、`理`、`空`、`列`、`元`、`典`、`遍器`、`任务`、`法`、`类`、`协`、`模块`、`误`、`对象`与`任意`。类型可以形成联合`甲|乙`、可空`甲?`、容器泛型及法类型。`列<T>`和`典<K,V>`递归约束已有与新增元素；`元<T…>`按位置和长度约束；`任务<T>`表示等候结果。

未标注位置允许渐进动态值。显式标注必须在静态可确定时报告不兼容，并在运行时赋值、传参和返回边界再次验证。`任意`接受全部值，但不会取消后续显式边界。`值 是 类型`返回理值，运行时递归检查联合、可空与容器泛型，并识别完整类继承链和显式协议链。`若 变量 是 类型`在真假分支收窄简单变量；对`空`的相等或不等判断继续收窄可空联合。

类支持单继承、实例/静态域和方法。`公`、`私`控制外部可见性，`只`域只能初始化一次。初始化法不得为静法或异法。实例方法自动绑定`此`；方法查找沿父类进行，字段初值按父类至子类次序建立。子类实例可赋给任一祖先类标注。

子类实例法内可以调用`父.方法（…）`，包括显式`父.初始化（…）`。调用从当前方法定义类的直接父类开始查找实现，绑定的`此`仍是当前动态实例，因此多层继承中的父调用不会重新分配对象。类外、静法内、无父类时使用`父`，读取裸`父`，或调用不可见/不存在的父类法均为错误。

覆写必须保持成员种类、静法/实例法属性和完整参数/归值签名；公开父类法不得收窄为私有。继承域不得被重新声明，也不得改写为法。动态派发继续选择实际实例的最具体实现，`父.方法`是显式跳过该派发的唯一形式。

协议是结构要求。类以`纳`显式声明协议后，必须提供兼容的字段和方法；子类继承父类的协议符合关系。协议类型的变量可以接纳符合该协议的实例。1.x 不承诺多继承，也不承诺未显式`纳`的偶然结构自动转换。
