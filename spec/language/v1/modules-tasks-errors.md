# 模块、任务与错误

文件模块按确定性`ModuleId`缓存，每个运行时中至多初始化一次。包内模块使用包身份与包内相对路径，标准模块使用标准命名空间，无包文件才退化为规范文件身份。只有`公`声明进入导出表；未公开成员不得从外部读取或改写。相对导入相对于当前模块，`标准:名`选择标准模块，`包:名`通过清单与锁文件解析。经过锁定与内容校验的包源码根只获得模块读取资格，不因此获得普通文件或其他宿主能力。循环导入必须报告完整链路。

顶层`公 引「模块.yx」为 名；`把该模块命名空间公开重导出。消费者可继续访问`入口.名.类型`，多层 facade 可形成任意深度路径。重导出保留原声明的规范类型身份，不创建包装类型，也不把普通值绑定当作类型别名。重导出链中的每一段都必须公开；循环、同名导出冲突和访问未公开成员必须产生确定性诊断。

`异 法`调用产生惰性的`任务<T>`。第一次`候`把状态从`待行`推进至`运行`，完成或失败后缓存结果。重复等候完成/失败任务返回相同结果/错误；取消只影响待行任务。任务不得在运行中等候自身。`并候`按输入顺序推进任务；任一失败时，必须取消同组尚未开始的任务并增加结构化并候踪迹。

1.x 任务是单线程、确定性的协作模型，不承诺抢占或线程并行。该保证属于可观察执行顺序的一部分。

`抛`把普通值转换为误，或保留既有误的代码、类别、消息、位置和踪迹。`试…救`只捕获运行错误；词法、语法、名称解析、静态类型和编译错误发生在执行前。运行错误必须优先定位最内层失败表达式，并按调用、模块或结构化任务边界增加踪迹。

捕获后的误提供只读`代码`、`类别`、`消息`、`位置`和`踪迹`。普通运行错误的代码为`RUN000`、类别为`运行`；网络错误类别为`网络`，代码使用稳定的`NET_`前缀；套接字错误类别为`套接字`，代码使用稳定的`SOCKET_`前缀；字节转换与密码错误类别为`字节`，代码使用稳定的`BYTES_`前缀。稳定网络代码包括`NET_PERMISSION`、`NET_HEADER`、`NET_URL`、`NET_DNS`、`NET_CONNECT`、`NET_TLS`、`NET_TIMEOUT`、`NET_WRITE`、`NET_READ`、`NET_LIMIT`、`NET_PROTOCOL`、`NET_STATUS`和`NET_UTF8`。稳定套接字代码包括`SOCKET_PERMISSION`、`SOCKET_ADDRESS`、`SOCKET_DNS`、`SOCKET_CONNECT`、`SOCKET_BIND`、`SOCKET_ACCEPT`、`SOCKET_TIMEOUT`、`SOCKET_READ`、`SOCKET_WRITE`、`SOCKET_LIMIT`、`SOCKET_STATE`、`SOCKET_EOF`、`SOCKET_UTF8`和`SOCKET_UNSUPPORTED`。代码用于程序判断，消息可在不改变错误语义时改进。

`字节串`是不可变的任意二进制值，按内容相等，不提供源码字面量。`标准:字节`是构造边界：`从文字/转文字`执行 UTF-8 编解码，`长度`按字节计数，`切片`使用半开范围，另有`拼接/查找/从数列/转数列`。单个字节串的运行时宿主硬上限为 16 MiB，CLI、YXB 和嵌入入口都不能放宽；超限使用`BYTES_LIMIT`。无效 UTF-8 必须报`BYTES_UTF8`，不得替换或丢弃字节。

原生`网络`模块支持`http`和经根证书验证的`https`，并解码 HTTP/1.1 分块正文。`获取/发文`的全程超时默认为 10 秒、最大响应为 4 MiB；`请求（方法，地址，正文，超时毫秒，最大字节）`允许显式正整数预算并返回含`状态/地址/首部/正文`的典。`请求字节（方法，地址，首部，正文，超时毫秒，最大字节）`接受`典<文，文>`首部和可空字节正文，并把响应正文保留为字节串；实现必须拒绝 CRLF、非法首部和调用方控制的 Host、长度及逐跳首部。实际累计响应预算取应用请求值与宿主 HTTP/字节值上限的最小值，默认宿主上限为 16 MiB；它不是套接字单次读取上限。DNS、连接、TLS、重定向和正文传输共用同一调用级截止时间；每一跳都必须重新校验权限，并在 DNS 后对所有真实 IP 复查。已知长度和流式正文超限都必须以`NET_LIMIT`失败，不得把截断正文作为成功结果。WASI 不隐式获得网络能力。

原生`套接字`模块提供 TCP 流与监听器、UDP 数据报及地址查询。`TCP连接（地址，超时毫秒）`、`TCP监听（地址）`、`接受（监听器，超时毫秒）`、`发送（流，正文，超时毫秒）`、`接收（流，最大字节，超时毫秒）`、`UDP绑定（地址）`、`UDP发送至（套接字，正文，地址，超时毫秒）`和`UDP接收自（套接字，最大字节，超时毫秒）`构成文字入口。二进制入口包括`发送字节/接收字节/精确读取/UDP发送字节至/UDP接收字节自`；`接收字节`返回`数据/已结束`，零长度读取只在`已结束=真`时表示 EOF，`精确读取`提前结束必须报`SOCKET_EOF`。`关闭写端`执行 TCP 半关闭，`TCP无延迟`显式控制 Nagle 策略。超时须在 1..=86,400,000 毫秒，单次读取的运行时宿主硬上限为 4,194,304 字节，所有执行入口都不能放宽；该上限独立于 HTTP 累计响应预算。超限或非完整 UTF-8 不得返回截断成功值。`关闭`幂等，关闭后的资源操作报`SOCKET_STATE`。TCP 是无消息边界字节流；`接收`可在同一调用中继续读取以补全已到达但被分片的 UTF-8 字符，协议分帧仍由调用方负责。每个运行时同时开放的套接字和监听器受宿主资源上限约束。WASI 不隐式获得原生套接字能力。

`文件.读取字节/写入字节/追加字节`保留任意二进制，`文件.状态`至少返回`种类/字节数/只读/修改毫秒`。`随机.安全字节`使用操作系统安全随机源；`哈希.HMACSHA256`返回 32 字节摘要，`哈希.恒时相等`避免按首个差异提前返回。`环境.参数`返回运行器传入的用户参数，`日期.HTTP日期/解析HTTP日期`使用 HTTP IMF-fixdate。
