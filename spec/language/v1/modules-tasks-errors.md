# 模块、任务与错误

文件模块按规范绝对路径缓存，每个运行时中至多初始化一次。只有`公`声明进入导出表；未公开成员不得从外部读取或改写。相对导入相对于当前模块，`标准:名`选择标准模块，`包:名`通过清单与锁文件解析。循环导入必须报告完整链路。

`异 法`调用产生惰性的`任务<T>`。第一次`候`把状态从`待行`推进至`运行`，完成或失败后缓存结果。重复等候完成/失败任务返回相同结果/错误；取消只影响待行任务。任务不得在运行中等候自身。`并候`按输入顺序推进任务；任一失败时，必须取消同组尚未开始的任务并增加结构化并候踪迹。

1.x 任务是单线程、确定性的协作模型，不承诺抢占或线程并行。该保证属于可观察执行顺序的一部分。

`抛`把普通值转换为误，或保留既有误的消息、位置和踪迹。`试…救`只捕获运行错误；词法、语法、名称解析、静态类型和编译错误发生在执行前。运行错误必须优先定位最内层失败表达式，并按调用、模块或结构化任务边界增加踪迹。
