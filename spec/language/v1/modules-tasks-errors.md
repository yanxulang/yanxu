# 模块、任务与错误

文件模块按规范绝对路径缓存，每个运行时中至多初始化一次。只有`公`声明进入导出表；未公开成员不得从外部读取或改写。相对导入相对于当前模块，`标准:名`选择标准模块，`包:名`通过清单与锁文件解析。循环导入必须报告完整链路。

`异 法`调用产生惰性的`任务<T>`。第一次`候`把状态从`待行`推进至`运行`，完成或失败后缓存结果。重复等候完成/失败任务返回相同结果/错误；取消只影响待行任务。任务不得在运行中等候自身。`并候`按输入顺序推进任务；任一失败时，必须取消同组尚未开始的任务并增加结构化并候踪迹。

1.x 任务是单线程、确定性的协作模型，不承诺抢占或线程并行。该保证属于可观察执行顺序的一部分。

`抛`把普通值转换为误，或保留既有误的代码、类别、消息、位置和踪迹。`试…救`只捕获运行错误；词法、语法、名称解析、静态类型和编译错误发生在执行前。运行错误必须优先定位最内层失败表达式，并按调用、模块或结构化任务边界增加踪迹。

捕获后的误提供只读`代码`、`类别`、`消息`、`位置`和`踪迹`。普通运行错误的代码为`RUN000`、类别为`运行`；网络错误类别为`网络`，代码使用稳定的`NET_`前缀；套接字错误类别为`套接字`，代码使用稳定的`SOCKET_`前缀。稳定网络代码包括`NET_PERMISSION`、`NET_URL`、`NET_DNS`、`NET_CONNECT`、`NET_TLS`、`NET_TIMEOUT`、`NET_WRITE`、`NET_READ`、`NET_LIMIT`、`NET_PROTOCOL`、`NET_STATUS`和`NET_UTF8`。稳定套接字代码包括`SOCKET_PERMISSION`、`SOCKET_ADDRESS`、`SOCKET_DNS`、`SOCKET_CONNECT`、`SOCKET_BIND`、`SOCKET_ACCEPT`、`SOCKET_TIMEOUT`、`SOCKET_READ`、`SOCKET_WRITE`、`SOCKET_LIMIT`、`SOCKET_STATE`、`SOCKET_UTF8`和`SOCKET_UNSUPPORTED`。代码用于程序判断，消息可在不改变错误语义时改进。

原生`网络`模块支持`http`和经根证书验证的`https`，并解码 HTTP/1.1 分块正文。`获取/发文`的全程超时默认为 10 秒、最大响应为 4 MiB；`请求（方法，地址，正文，超时毫秒，最大字节）`允许显式正整数预算并返回含`状态/地址/首部/正文`的典。DNS、连接、TLS、重定向和正文传输共用同一调用级截止时间；每一跳都必须重新校验权限，并在 DNS 后对所有真实 IP 复查。超限必须以`NET_LIMIT`失败，不得把截断正文作为成功结果。WASI 不隐式获得网络能力。

原生`套接字`模块提供 TCP 流与监听器、UDP 数据报及地址查询。`TCP连接（地址，超时毫秒）`、`TCP监听（地址）`、`接受（监听器，超时毫秒）`、`发送（流，正文，超时毫秒）`、`接收（流，最大字节，超时毫秒）`、`UDP绑定（地址）`、`UDP发送至（套接字，正文，地址，超时毫秒）`和`UDP接收自（套接字，最大字节，超时毫秒）`构成稳定入口。超时须在 1..=86,400,000 毫秒，单次读取须在 1..=4,194,304 字节；超限或非完整 UTF-8 不得返回截断成功值。`关闭`幂等，关闭后的资源操作报`SOCKET_STATE`。TCP 是无消息边界字节流；`接收`可在同一调用中继续读取以补全已到达但被分片的 UTF-8 字符，协议分帧仍由调用方负责。每个运行时同时开放的套接字和监听器受宿主资源上限约束。WASI 不隐式获得原生套接字能力。
