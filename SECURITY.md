# 安全策略

## 支持范围

言序 1.x 已进入稳定支持期。安全修复优先提供给最新稳定版本；影响公开 1.x 语义、嵌入边界、包格式或宿主权限的高严重度问题，必要时会回补仍受支持的最近一个次版本。

尚未发布的开发分支只接受前向修复，不承诺单独维护。已结束支持的 0.x 版本应升级到最新 1.x；安全修复可以拒绝此前会导致越权、资源失控或归档欺骗的输入，并会在变更记录中说明兼容影响。

## 运行时安全边界

1.1.3 起，网络外连、TCP 监听和 UDP 绑定为三项独立能力；沙箱默认全部拒绝。HTTP 每次重定向都重新校验主机，外连在 DNS 解析后复查真实 IP。`*`不会隐式授权敏感目标；回环、私网、链路本地和 CGNAT 地址须按 IP 精确授权，或由应用与宿主共同授予 1.1.15 新增的“本地网络”能力。未指定、广播、组播、文档和其他保留地址仍只接受精确 IP。

每个运行时最多持有 128 个套接字资源和 16 个 TCP 监听器。远程包制品只解压普通文件与目录，不接受链接、设备、管道或越界路径，并对压缩与展开资源设置硬上限。

1.1.7 的 GUI 能力、剪贴板、文件对话框、系统通知、托盘、外部地址和全局快捷键彼此独立，且不会提升文件、网络、进程或普通原生扩展权限。ABI v2 对递归值、队列、回调和父子资源设置硬上限与代际检查；后台线程只能投递，不能直接进入 VM。YXB 与 Bundle 的动态库须匹配锁定目标、ABI、大小和 SHA-256，并从私有内容寻址只读路径装载。原生库仍是进程内受信任代码，摘要不等同于发布者签名。完整威胁模型见[`reference/gui-security.md`](reference/gui-security.md)。

## 报告漏洞

请不要为尚未修复的安全问题创建公开 Issue。使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告功能，并提供：

- 受影响版本与操作系统；
- 最小复现文卷或命令；
- 可观察到的影响；
- 如已知，建议的修复方向。

维护者会尽快确认收到报告，在理解影响后给出修复与披露计划。请在公开披露前为修复预留合理时间。
