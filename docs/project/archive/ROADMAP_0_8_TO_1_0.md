# 言序 0.8–1.0 稳定化工作台账

> 归档日期：2026-07-18

> 目标版本：`1.0.0`

本文件保存 0.8–1.0 的历史设计决定、进度与验收证据，不再描述项目当前状态。0.3–0.7 记录见[`DEVELOPMENT_0_3_TO_0_7.md`](DEVELOPMENT_0_3_TO_0_7.md)，当前版本与格式见[`../status.md`](../status.md)。

## 共同退出规则

- 功能必须同时具备语义说明、正向测试、错误测试、回归测试和可运行示例。
- 树解释器与 VM 的共享能力必须加入跨实现一致性套件，禁止任一执行器静默降级。
- 新语法必须同步格式化器、静态检查、LSP、文档生成、Tree-sitter、TextMate 与 VS Code 片段。
- 公共 API、清单和字节码格式必须带显式版本；破坏性变化必须进入迁移工具与兼容说明。
- 每个里程碑必须通过格式化、全部测试、严格 Clippy、示例回归和 Cargo 发布包隔离构建。
- 1.0 只在核心、官网、文档站和编辑器版本及措辞一致后完成；不以单纯修改版本号代替验收。

## 0.8：并发与宿主交互

### 任务与异步语义

- [x] `异 法`声明异步函数；调用后得到`任务<T>`而非立即执行正文。
- [x] `候`等待任务并取得结果；失败保持原始源码位置与调用踪迹。
- [x] `取消`和可观察任务状态；已完成任务可重复等候，运行中自等待有明确错误。
- [x] `并候`以结构化作用域等待任务列；任一失败时取消尚未开始的同组任务。
- [x] 树解释器、VM、静态类型、格式化器和工具链语义一致。

### 嵌入与权限

- [x] 稳定 Rust 嵌入 API：配置后端与权限，持久运行源码/文卷并取得输出与结构化错误。
- [x] C ABI 最小边界：创建/销毁运行时、执行 UTF-8 源码、取得/释放版本化 JSON 结果。
- [x] 包清单`[权限]`支持文件、网络、环境与进程能力；默认嵌入模式最小授权，CLI 保持显式兼容策略。
- [x] 文件模块、文件/环境/网络标准库在两套运行时和静态检查阶段执行同一权限判定。
- [x] WASM/WASI 入口和构建检查；不支持的宿主能力返回明确错误。
- [x] 将 0.9 的执行步数、调用深度与集合增长预算接入`EngineConfig`，完成嵌入执行预算配置。
