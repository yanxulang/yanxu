# 言序 0.3–0.7 完整性工作台账（已完成）

> 归档日期：2026-07-18
> 完成时发行号：`0.7.0`
> 最终结论：0.3–0.7 退出条件及发行后标准库复核均已完成。

本文件保存 0.3–0.7 的历史验收记录，不再描述项目当前状态。当前版本与格式见[`../status.md`](../status.md)。

## 稳定验收规则

- 每项语言能力至少有正向、错误与回归测试。
- 树解释器与 VM 共同支持的语句必须通过一致性测试。
- 新语法必须同步格式化器、Tree-sitter、TextMate、LSP 与文档。
- 每个里程碑必须通过 `cargo fmt`、全部测试、严格 Clippy 和发布打包。
- 不得用静默回退、高层 AST 直接执行或文档措辞代替未完成的 VM/工具实现。

## 工作日志

### 2026-07-14：重新审计

- 对照原路线和退出条件逐项复查。

- 后续依赖顺序：迭代协议 → 类型/对象 → 包/标准库 → 完整 VM → 语义工具。

### 2026-07-14：完成 0.3 退出条件

- 新增共享的惰性`遍器`状态；`逐`、`遍/续`、范围和数据管道不再各自特判容器。
- 用户对象可用`遍始/遍次`加入协议，`遍次`以`（是否尚有，值）`表达耗尽且不混淆`空`值。
- 新增惰性`映射/筛选`与`折叠/排序/反转/包含/寻找/范围/步进范围`。
- 新增正向、惰性、用户协议和错误回归测试，以及`examples/数据管道.yx`。
- 验证：49 项 Rust 测试通过，示例静态检查与实际执行通过。

### 2026-07-14：完成 0.4 退出条件

- `TypeRef`改为结构化类型树，规范支持`数?`、联合、`列<T>`、`典<K,V>`、定长元组与`法（...）：...`。
- 静态检查器现可推断容器元素/下标/法调用类型，并按`空`比较和`类型(...)`判定进行分支收窄。
- 新增`协/纳`、协议签名符合性、声明式`域`、`公/私/只/静`成员和静态法；运行时同步执行可见性、只读与字段类型防线。
- 跨模块检查读取公开 API 摘要并递归检查依赖，测试证明此过程不会执行模块顶层代码。
- 格式化器、文档生成器和 Tree-sitter 已同步新语法并重新生成解析器。
- 验证：57 项 Rust 测试、严格 Clippy 和 Cargo 发布打包通过。

### 2026-07-14：完成 0.5 退出条件

- 清单改用完整 TOML 模型（兼容中文裸键），严格校验包名、包内入口、SemVer 和版本要求。
- `言序.lock`固定来源、版本、Git 精确提交、入口和目录 SHA-256；内容被改写时拒绝静默更新，`包 更新`用于显式重锁。
- 路径、Git、`file://`/远程中央索引使用统一解析器；Git 与索引缓存支持离线验证，远程制品下载前后均有路径/内容校验。
- 模块加载器记录确定的初始化顺序，循环诊断显示完整`甲 → 乙 → 甲`链。
- 标准库新增文字、扩展数学、时间、文件、JSON、HTTP 网络和测试模块；I/O、UTF-8、JSON 类型及网络状态均有结构化运行错误。
- 新增`examples/标准库.yx`；验证：62 项 Rust 测试、示例静态/运行检查、严格 Clippy 和发布打包通过。

### 2026-07-14：完成 0.6 退出条件

- 字节码不再保留 AST 或回退到树解释器；函数、闭包、类、协议、异常、模块和迭代均降为独立原型及指令。
- VM 新增显式调用帧、词法环境、递归闭包、方法绑定、继承、静态/只读/私有字段和结构协议检查。
- `逐`、惰性遍器、典、切片、字段/下标改写、结构化错误和核心内建法均由 VM 自身执行。
- GC 公开分配/存活/回收统计，并用不可达自引用闭包验证环回收；属性缓存和模块字节码缓存分别验证命中及源文件元数据失效。
- 新增跨实现一致性规格，覆盖控制流、闭包递归、容器、惰性迭代、对象协议、异常与相对模块；该规格发现并修复了 VM `反转`错误排序的问题。
- 新增`yanxu 基准 [轮数]`，分开报告解析、编译、树解释器和 VM 执行耗时，并先校验两端输出相同。
- 验证：69 项 Rust 测试、严格 Clippy 和 Cargo 发布打包通过。

### 2026-07-14：完成 0.7 退出条件

- 新增可复用语义索引，LSP 现以同一词法作用域模型提供语义补全、定义、引用、重命名、悬停与层级文档符号，并正确换算 UTF-16 坐标。
- 参考解释器新增只读执行前调试快照；`yanxu 调试服务`以 DAP 核心消息提供源码断点、继续、单步、调用栈、作用域变量与悬停求值，内存协议测试覆盖完整暂停会话。
- 测试运行器新增路径过滤、有界并发、逐卷超时、`# 预败`/意外通过状态和带版本模式的 JSON 报告；CLI 可组合全部选项。
- 文档生成器读取`///`/`#:`声明注释，生成类型链接、显式稳定锚点和模块 API 索引；目录输入可汇总多模块文档。
- REPL 接入终端行编辑器，方向键历史、Tab 会话补全、Ctrl-C、多行验证和历史持久化已在真实伪终端中逐键验收。
- 验证：77 项 Rust 测试、`cargo fmt --check`和严格 Clippy 通过；下一步按既定门槛同步官网、下载、文档和编辑器仓库。

### 2026-07-14：完成发行后复核与标准库扩建

- 复核发现 VM 的标准模块适配此前只覆盖数学三项，现已补齐文字、完整数学、时间、文件、JSON、网络与测试模块；读写、HTTP、JSON 和错误行为均加入树解释器/VM 一致性规格。

- 新增纯函数正向/错误单元测试、扩展模块类型测试、跨运行时一致性测试和`examples/扩展标准库.yx`。
- 验证：核心 66 项单元测试、15 项语言集成测试与 7 项双运行时一致性测试（共 88 项）通过；扩展示例静态检查及双引擎输出一致；严格 Clippy、Cargo 发布打包、文档站 71 页构建、官网构建与 VS Code 扩展预发布全部通过。
