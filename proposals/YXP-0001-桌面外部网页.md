# YXP-0001：桌面外部网页

- 状态：接受
- 目标版本：1.2.0

## 摘要

新增`标准:桌面.打开网页（地址：文）：空`，让图形应用在运行时把 HTTP(S) 地址交给系统默认浏览器。

## 动机

包清单和宿主权限协议已经定义`打开外部地址`，但言序标准库没有消费该权限的公共 API。应用只能依赖特定 UI 宿主、静态链接或申请过宽的通用进程权限，无法安全打开用户配置的服务地址。

## 用户示例

```yanxu
引「标准:桌面」为 桌面；
桌面.打开网页（「https://example.com」）；
```

清单必须声明`打开外部地址 = true`。

## 词法、文法与静态类型

不增加词法或文法。标准模块`桌面`新增一元法`打开网页`，参数为`文`，返回`空`。

## 运行语义与错误

调用先核验`打开外部地址`权限，再修剪并解析 URL。只接受含主机的`http`或`https`地址。macOS、Windows、Linux 分别调用系统默认网页启动器。错误码为`DESKTOP_URL`、`DESKTOP_OPEN`和`DESKTOP_UNSUPPORTED`；权限拒绝使用既有权限诊断。

## 权限与资源预算

能力不隐式授予通用进程、文件或网络权限。`file:`等 scheme 被拒绝。调用同步等待系统启动器退出，但不等待浏览器或页面加载；不捕获输出，资源开销有固定上界。

## 双执行器与工具链

树解释器使用受权限保护的 intrinsic，VM 使用对应`StandardNative`，两者复用`stdlib::open_external_http_url`。静态类型摘要、机器 API 清单、LSP 标准库成员、WASI 不支持诊断与三平台构建同步更新；格式化器、Tree-sitter 和 VS Code 无语法变化。

## 兼容、迁移与文档

这是向后兼容的标准库新增能力，按 SemVer 进入 1.2.0。既有源码、YXB、锁文件和原生 ABI 不变。需要该能力的应用将最低言序版本提高到 1.2.0，并声明现有权限即可。

## 测试与安全

测试覆盖公网/回环 HTTP(S)、非 HTTP scheme、非法地址、权限先于 URL 校验、树解释器/VM 一致性、标准库清单与兼容语料。拒绝从 URL 启动任意命令，不允许本地文件 scheme，不把地址交给 shell 解析。

## 替代方案与未决问题

拒绝使用`进程.执行（open/xdg-open）`，因为它要求过宽权限；拒绝加入特定 UI 库，因为能力属于运行时。暂不支持`mailto:`等额外 scheme，后续扩展需单独提案。无未决问题。
