# 迁移到 1.1.20

言序 1.1.18 与 1.1.19 均未形成稳定 Release；使用 1.1.17 的项目应直接升级到 1.1.20。1.1.20 正式交付依赖不可变快照、归档、原生装载、审计、解析深度限制和供应链门禁，详细边界见[`migration-1.1.18.md`](migration-1.1.18.md)与[`migration-1.1.19.md`](migration-1.1.19.md)。清单格式、锁文件格式、YXB、字节码、原生 ABI 和工程协议版本均未改变。

1.1.19 预发布格式化器会把单元素元组`（值，）`改写为普通括号表达式`（值）`。如果源码曾由该预发布版本执行写入式格式化，请在升级后：

- 使用版本控制检查 1.1.19 格式化产生的差异；
- 为仍表示元组的单元素括号表达式恢复末尾逗号；
- 重新运行`yanxu fmt --write`两次并确认第二次不再产生变化；
- 重新执行类型检查和测试，确认元组与普通括号表达式的预期类型未被混淆。

未使用 1.1.19 预发布写入式格式化的项目无需修改源码。1.1.20 会保留单元素元组的消歧逗号，重复格式化结果稳定一致。
