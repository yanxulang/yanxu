# 迁移到模块限定类型

模块限定类型保持当前模块裸类型语法，但移除一个未规定的旧行为：检查器曾因共享
全局短名称表，使某些已导入类型在消费模块中偶然可以裸写。该行为依赖检查顺序，
也会让两个模块的同名类型发生碰撞。

## 改写导入类型

旧代码可能偶然通过检查：

```yanxu
引「base.yx」为 基础；
定 根：视图 为 基础.视图（）；
```

现在应把导入别名写入类型访问路径：

```yanxu
引「base.yx」为 基础；
定 根：基础.视图 为 基础.视图（）；
```

这项改写适用于域、参数、归值、局部绑定、遍历变量、联合、可空、泛型、法类型、
`承`、`纳`和`是`。例如：

```yanxu
类 按钮 承 基础.视图 纳 基础.可描述 则
    域 父项：基础.视图?；
    法 包装（项目：列<基础.视图>）：基础.视图 则
        归 项目【0】；
    终
终
```

当前模块声明继续裸写：同文件中的`视图`无需改为限定路径。普通值访问
`基础.函数`和`基础.常量`也不改变。

## 导入别名与同名类型

别名不是类型身份。同一模块分别导入为`基础`和`核心`时，两条路径解析到同一
`TypeId`。不同模块都导出`节点`时，必须分别写`甲.节点`和`乙.节点`，两者保持
不同身份。

## 公开入口模块

包入口可以直接公开内部模块：

```yanxu
公 引「base.yx」为 基础；
公 引「controls.yx」为 控件；
```

消费者随后使用`界面.基础.视图`或`界面.控件.按钮`。重导出不创建包装类，原类
的规范身份保持不变。`公 引`只能位于顶层。

## 工具与制品

- 运行`yanxu fmt --write`可稳定格式化限定类型，但不会猜测并自动插入导入别名；
- 外部限定类型的 LSP definition/hover 会跟随重导出到原声明；导入别名可在当前
  文档安全 rename；
- `yanxu 文 --json`现在输出[模块 API 清单 v2](module-api-v2.md)；
- 字节码格式已升级到 2，旧 YXB 或缓存制品须从源码重新构建。

完整可运行示例位于[`examples/module_oop`](../examples/module_oop)。
