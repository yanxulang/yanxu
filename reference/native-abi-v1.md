# 言序原生扩展 ABI v1

原生 ABI v1 用于把 GUI、数据库、音频、图像与系统 API 等宿主能力封装为独立包。ABI 边界只使用 C 兼容布局、固定宽度数字、指针/长度对和 UTF-8 JSON，不暴露 Rust 类型布局。规范头文件是[`include/yanxu_native.h`](../include/yanxu_native.h)，可运行参考实现是[`examples/native-extension-rust`](../examples/native-extension-rust)。

## 安全门禁

言序只有在以下条件全部成立时才打开动态库：

1. 顶层应用显式声明`[权限].原生扩展 = true`；
2. 锁定制品的 ABI 必须精确为 1，目标与`yanxu version --json`报告的目标完全相同；
3. 锁定大小必须与同一打开句柄的实际大小和读取长度一致；制品须为小于等于 256 MiB 的普通文件，不得为符号链接、重解析点、FIFO、设备或目录，字节 SHA-256 与清单/锁文件中的 64 位十六进制校验和一致；
4. 动态库导出`yanxu_native_module_v1`，描述符的 ABI 版本、精确结构大小、模块名、数组长度、指针、UTF-8 和名称唯一性全部通过校验。

校验通过后，运行时不会再按原路径打开动态库，而是把已校验字节写入进程私有、以 SHA-256 命名的只读目录，使用同一暂存文件句柄复核副本摘要，并将该句柄保持到动态库卸载。Linux/macOS 的目录从`0700`创建、制品从`0600`创建，复核后分别收紧为`0500`和`0400`；Windows 同样使用随机私有目录和只读制品，暂存句柄只共享读取且拒绝写入和删除，并在清理前先卸载 DLL。macOS 的系统代码签名/公证策略与此摘要门禁叠加生效，运行时不会绕过系统加载策略。原路径在校验后被替换不会改变本次装载内容。

旧锁定条目若没有记录真实制品大小，须从受信来源重新取得制品并执行`yanbao install`生成新锁文件；不要手工补写大小或摘要。符号链接、重解析点和特殊文件不提供兼容回退，必须改为包内真实普通文件。

原生动态库属于与宿主进程同权限的受信任代码，能够直接调用操作系统 API，不受言序文件、网络、环境或进程沙箱完全约束。`原生扩展`权限、目标/摘要校验和只读暂存用于缩小供应链与替换风险，不构成对恶意原生代码的隔离。只应装载经过审核、来源可验证的扩展。

WASI 目标始终返回`NATIVE_UNSUPPORTED`，不会尝试打开宿主动态库。言序不执行第三方包的构建脚本、安装钩子或 Shell 命令。

## 清单声明

```toml
[包]
格式 = 2
名 = "图像扩展"
版 = "1.0.0"
言序 = ">=1.1.6"
入口 = "src/图像.yx"

[原生]
ABI = 1

[原生.macos.aarch64]
文件 = "native/aarch64-apple-darwin/libimage.dylib"
校验和 = "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"

[原生.linux.x86_64]
文件 = "native/x86_64-unknown-linux-gnu/libimage.so"
校验和 = "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"
```

解析器只选择当前目标的一个制品，并把目标、相对路径和校验和固定到锁文件。引用原生包的应用仍必须在自己的`权限`表开启`原生扩展`；依赖不能传递授权。

## 模块描述符

`YanxuNativeModuleV1.struct_size`必须精确等于当前规范头文件中`YanxuNativeModuleV1`的尺寸。ABI v1 不通过在结构尾部追加字段来扩展；需要新布局时必须使用新的 ABI 版本和入口符号。使用官方头文件或参考实现并写入`sizeof(YanxuNativeModuleV1)`的既有扩展无需迁移；曾写入自定义更大或更小尺寸的扩展必须按当前 v1 头文件重新构建。

`YanxuNativeModuleV1`注册以下公开面：

- 函数：收取一个 UTF-8 JSON 参数值，返回 JSON 值或不透明资源；
- 常量：描述符存续期内稳定的 UTF-8 JSON 值；
- 资源类型：全局唯一类型名，返回资源时必须同时给出非空释放函数；
- 回调：宿主通过`YanxuNativeCallbackV1`把命名回调和 JSON 参数传回言序运行时；
- 错误：非`YANXU_NATIVE_OK`状态必须返回稳定代码和面向用户的 UTF-8 消息。

函数输出由模块的`free_bytes`释放；不透明资源由单独的`drop_resource`释放。言序从函数返回起使用 RAII 守卫接管输出，因此失败状态、空指针、异常长度、JSON 解析失败、未注册资源类型和未知输出种类也会走可用的释放器。言序在资源存活期内保持动态库已加载，且每个资源只释放一次。单次 JSON 输入/输出硬上限为 16 MiB，每类描述符最多 1024 项；模块/函数/资源名称、错误码和错误消息另有较小上限。

## 能力查询与验证

```sh
yanxu native --json
cargo test --test native_extension
```

`能力查询`只描述当前运行时支持的 ABI 面，不授予加载权限。对外发布的制品应在 Windows、Linux 和 macOS 对其声明目标分别构建和校验。
