# 迁移到言序 1.1.16

1.1.16 不改变语言语法、标准库签名、清单、锁文件、YXB、当前字节码、原生 ABI 或工程协议版本。既有 1.x 源码、格式 2 锁文件和字节码格式 2 制品可直接使用。

## 二进制资源预算

单个二进制值和 HTTP 响应正文受 16 MiB 宿主硬上限约束。HTTP 默认累计预算仍为 4 MiB，显式请求值与宿主上限取较小值；套接字单次读取继续独立限制为 4 MiB。需要处理更大正文的应用应采用流式协议或应用层分块，不能通过嵌入配置放宽硬上限。

超限读取以`NET_LIMIT`失败，不会返回截断后仍标记成功的响应。精确等于有效预算的正文仍然成功。

## 旧 YXB 与字节码

外层 YXB 格式 1 仍受支持，但其中字节码必须为格式 2。字节码格式 1 不含无歧义恢复模块与类型身份所需的信息，不能自动迁移；请保留原源码并执行：

```sh
yanxu compile <源码或项目> -o <新制品.yxb> --release
```

诊断中的`BYTECODE_FORMAT_UNSUPPORTED`或`YXB_BYTECODE_UNSUPPORTED`会列出检测格式、当前格式、不可安全迁移结论和同一重建命令。损坏、越界或摘要不符的归档仍按安全错误拒绝，不进入兼容路径。
