# 迁移到 1.1.18

1.1.18 不改变清单或锁文件格式。依赖解析的安全边界收紧：路径、Git 和注册表依赖通过清单、锁定目标和完整树摘要校验后，会发布到`YANXU_CACHE`下内容寻址、只读的`resolution/tree-v1` generation；解释器、VM、类型检查、应用构建、LSP 和文档工具消费解析阶段保存的同一目录能力，不再根据来源路径重新打开已经校验过的树。

进程内依赖图缓存每次命中都会复验应用根身份、清单、锁文件、当前目标、来源树摘要和 generation 摘要。来源目录、同长文件或锁文件在校验后发生变化时，旧 generation 保持不变，当前操作会拒绝继续；请审核变化后显式执行`yanbao update`或`yanbao install`生成新的锁定结果，不要手工改写摘要。

工程协议中的依赖`root`和`entry`现在指向实际执行的不可变 generation，不能再用它们推断路径依赖的作者工作目录。来源种类和声明值继续由锁文件`source`字段及工程协议的来源字段表达。删除 generation 缓存不会改变锁定结果；只要原始来源或经过校验的 Git/注册表缓存仍可用，下一次解析会按同一摘要重建。
