# 迁移到 1.1.18

1.1.18 不改变清单或锁文件格式。依赖解析的安全边界收紧：路径、Git 和注册表依赖通过清单、锁定目标和完整树摘要校验后，会发布到`YANXU_CACHE`下内容寻址、只读的`resolution/tree-v1` generation；解释器、VM、类型检查、应用构建、LSP 和文档工具消费解析阶段保存的同一目录能力，不再根据来源路径重新打开已经校验过的树。

包元数据读取现在统一拒绝最终文件符号链接、最终文件重解析点、目录、管道和读取期间发生身份替换的文件，并执行 4 MiB 清单、16 MiB 锁文件、8 MiB 辖制清单和 32 MiB 注册表索引上限；工作区成员清单适用同一 4 MiB 上限。超过上限的既有文件应删除重复或非必要字段后重试，不能依靠提高进程内存继续解析。远程索引只缓存已完成有界读取、重复键检查和结构校验的同一份字节。

进程内依赖图缓存每次命中都会复验应用根身份、清单、锁文件、当前目标、来源树摘要和 generation 摘要。来源目录、同长文件或锁文件在校验后发生变化时，旧 generation 保持不变，当前操作会拒绝继续；请审核变化后显式执行`yanbao update`或`yanbao install`生成新的锁定结果，不要手工改写摘要。

工程协议中的依赖`root`和`entry`现在指向实际执行的不可变 generation，不能再用它们推断路径依赖的作者工作目录。来源种类和声明值继续由锁文件`source`字段及工程协议的来源字段表达。删除 generation 缓存不会改变锁定结果；只要原始来源或经过校验的 Git/注册表缓存仍可用，下一次解析会按同一摘要重建。

Git 依赖缓存迁移到`YANXU_CACHE/git/v2/<来源SHA-256>`：每个来源使用加锁的 bare 对象库，每个 40 位精确提交发布独立、只读的`tree-v1` generation。旧的共享可变 checkout 不再作为可信输入；首次升级后请联网执行一次`yanbao install`或`yanbao update`填充新缓存。离线解析只接受锁文件中的 40 位精确提交，不再根据分支、标签或其他符号修订切换共享工作树。Git 包的内容来自该提交的确定性归档；依赖外部 smudge 过滤器或未提交 Git LFS 内容的项目，应先把运行所需字节纳入可复现的包制品。

旧注册表目录`YANXU_CACHE/registry/<索引身份>/<包>/<版本>`不再直接进入依赖解析。锁文件含有效内容 SHA-256 时，1.1.18 会在对应包版本锁内复验旧目录，把同一内容捕获并原子发布到`.snapshots/tree-v1`下的只读 generation，然后只消费 generation；旧目录会保留，不会在迁移或失败恢复时被删除。迁移成功后可按普通缓存维护策略清理旧目录；迁移失败时请保留诊断并联网重新执行`yanbao install`，不要手工把旧目录移动到`.snapshots`。

工程协议仍使用版本 1，但`handshake`现在通过`operation_capabilities.audit`声明审计能力 Schema 1 和十二项稳定检查标识；`audit`响应以`audit_capabilities`逐值回显同一能力。审计消费端应先验证 Schema 与所需检查覆盖，并忽略未知的扩展操作或检查；缺少能力字段的旧核心不能被当作完成了全量安全审计。

`vendor`不再先删除目标目录。新内容会在同一父目录完成快照、校验和文件同步，再由事务替换既有的有效辖制目录；Unix 同步父目录项，Windows 使用写穿透目录改名。普通目录、链接、重解析点、内容损坏或含额外文件的目标都拒绝覆盖。发布或父目录同步失败会复验并恢复旧目录；若文件系统同时拒绝恢复或清理，错误会列出保留的隐藏 staging/backup 路径，不应在确认内容前手工删除它们。

`yanxu 试 --超时/--timeout`现在把时限绑定到解释器执行预算，不再仅停止等待并遗留后台用例线程。达到时限后，解释器会在受计量的执行点终止并完成栈展开；固定数量的测试工作线程全部退出后命令才返回，因此连续超时不会突破`--并发/--jobs`上限。若用例停在尚未返回的宿主操作中，命令会等待该操作结束后再报告超时，不会在操作仍运行时提前返回。

模块源码现在统一限制为 8 MiB（8,388,608 个 UTF-8 字节），恰好达到上限仍可读取，超限统一报告`PACKAGE_MODULE_SOURCE_LIMIT`。该限制覆盖解释器、VM、类型检查、应用构建、CLI、LSP、文档、测试、兼容检查及所有嵌入入口；此前依赖超大单文件的项目应在升级前拆分模块，不能通过更换入口绕过限制。

`yanxu 试`、兼容语料检查和目录文档生成现在从发现到读取始终持有同一工具目录根能力，并把该能力传递给顶层模块的相对导入。目录或同名模块在发现后被替换时，操作继续读取已发现的原根对象或明确失败，不会混入替换目录内容，替换目录中新增的更深包清单也不能改变根选择；符号链接、Windows 重解析点和特殊文件保持拒绝。目录工具不再隐式信任包外绝对路径或父目录越界导入；确需跨包共享的模块应声明为依赖并通过公开`包:`导出访问。工具遍历新增单目录 4,096 项、整树 100,000 项和相对根目录 128 层上限；超过预算的测试或文档树应拆分为多个显式入口后分别运行。

目录工具需要为`包:`导入重新解析依赖时，会在解析前后复验发现阶段的包根身份；根对象发生漂移会明确失败，不会改用同名替换目录的清单或锁文件。
