# GUI 安全与信任边界

原生 ABI 模块是进程内受信任代码。言序的权限和摘要门禁可以阻止意外越权、替换、
路径逃逸与资源耗尽，但不能隔离恶意动态库的任意系统调用。只应安装来源明确、锁定且
经过审计的 GUI 包；WASI 目标完全禁止动态库。

## 权限最小化

`图形界面`、`剪贴板`、`文件对话框`、`系统通知`、`托盘`、`打开外部地址`和
`全局快捷键`彼此独立。GUI 权限不会授予文件根、网络、进程或原生扩展权限；
文件对话框仅允许一次用户主动选择，取消返回空值。YXB 记录应用申请，实际权限始终是
申请与嵌入宿主上限的交集。任何包含动态库的 GUI 应用（包括`yanxu-gui`）都必须在
顶层【权限】中显式设置`原生扩展 = true`；包名、来源和 ABI 版本都不能代替该授权。

## FFI 与资源限制

ABI v2 拒绝无效 UTF-8、非有限浮点、错误指针/长度组合、过深值、过多元素、过大文字/
字节和过长错误输出。所有跨 FFI 调用使用 panic 隔离；模块错误与言序回调错误转为
结构化运行错误，Rust panic、C++ 异常和言序异常不得跨越 ABI 边界。

事件队列固定容量，高频连续事件合并；满队列、已退出队列、重入和错误线程都返回稳定
错误。回调和资源句柄含 generation，释放后使用不会落到新对象。父资源递归关闭子项，
析构 panic 被隔离，每项最多析构一次。

## 制品与路径

构建只接受锁文件选出的当前目标普通文件，拒绝符号链接、硬链接、设备、FIFO、特殊
文件、大小/数量/总量超限和摘要不符。YXB 解码限制压缩/展开大小、条目数、路径深度和
资源大小；原生模块解包进随机私有目录的内容寻址只读文件。Windows 使用受限 DLL
搜索标志。Bundle 以同级暂存、完整验证、原子替换和失败回滚避免半成品；启动时先验
外层清单和全部索引文件，再读取内嵌 YXB，外置动态库即使未被直接装载也不能被静默
篡改。

SHA-256 只证明当前字节与清单一致，不能证明发布者身份。正式分发仍应使用受保护标签、
审查过的 CI、平台代码签名/公证和独立校验文件。本次开发不自动创建标签或 Release。

## 故障排查

- `GUI_PERMISSION_*`：在应用清单申请对应能力，并确认嵌入宿主也授权。
- `GUI_WRONG_THREAD`或`GUI_RESOURCE_LOOP`：只在创建 VM 的线程操作 UI，后台仅 post。
- `GUI_RESOURCE_CLOSED`：父窗口、控件或事件循环已经关闭，不能继续保存并使用句柄。
- `GUI_QUEUE_FULL`：合并/限速生产者，不能通过无限重试形成忙循环。
- 原生摘要/目标错误：重新取得可信制品并显式运行`yanbao install`，不要手改锁文件。

安全问题请通过仓库 Security 私密报告渠道提交，避免在修复前公开可利用细节。
