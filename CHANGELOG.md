# 变更记录

## 1.1.20

### 格式化语义安全

- 格式化器现在保留单元素元组末尾用于消歧的逗号，`（值，）`不会再被改写成语义不同的普通括号表达式。
- 发布模糊测试发现的精确崩溃输入已固化为回归种子，并验证重复格式化保持幂等；原生扩展回归使用隔离构建目录，避免共享目标目录影响验收结果。

### 安全边界与发布门禁

- 1.1.19 预发布中已完成的解析深度限制、依赖不可变快照、归档与原生装载约束、Git 子进程预算、SBOM 和构建来源证明全部随本版本正式交付。
- 核心与共享包继续要求完整动态安全门禁；已有版本标签保持不可变，未通过门禁的预发布快照不会被移动或直接晋升为稳定版。

### 兼容与安装

- 语言语法、标准库签名、清单格式 2、锁文件格式 2、YXB 格式 1、字节码格式 2、原生 ABI v1/v2 和工程协议 1 均未提升版本。
- `v1.1.18`与`v1.1.19`均未形成稳定 Release；使用 1.1.17 的用户应直接升级到 1.1.20。曾使用 1.1.19 预发布格式化源码的用户应按迁移说明复核单元素元组逗号。

macOS 与 Linux 可固定安装本版本：

```sh

```

Windows PowerShell 可先设置`$env:YANXU_VERSION = "1.1.20"`，再运行
`irm https://get.yanxu.dev/windows | iex`。

### 包归档、应用资源与工作区

- 声明 Rust 1.89 为最低支持版本，并在当前/MSRV、六个原生目标和 WASI 门禁中核对；新增依赖漏洞、许可证、来源、重复依赖和 npm 高危审计。

- `audit`校验锁定校验和、来源、精确 Git 修订、许可证、漏洞、撤回状态、目标和原生制品属性，并以工程协议 Schema 1 声明十二项稳定检查能力。
- 依赖与公告来源拒绝嵌入凭据、敏感查询、片段、控制字符和歧义编码；诊断、工程 JSON、LSP 与外部命令错误统一避免回显敏感来源。

### Git 依赖子进程与缓存预算

- Git 提交检查、对象库初始化、远端抓取和精确归档分别使用 15 秒、30 秒、10 分钟和 2 分钟硬截止时间；超时、内部取消或输出超限会终止完整进程树并完成主进程回收，不再只结束直接 Git 进程。
- Git 标准输出和标准错误在持续排空时执行有界保留：无输出操作要求标准输出为空，提交探测最多 128 字节，标准错误最多 64 KiB；错误诊断不回显可能包含凭据的子进程输出。
- bare 对象库在命令运行前、运行中和发布前统一限制为 1 GiB、1,000,000 项和 256 层；初始化暂存目录限制为 4 MiB，提交归档继续受包树 640 MiB 总预算约束。磁盘扫描绑定已打开根目录身份并计入稀疏文件实际占用，根替换、链接、Windows 重解析点、特殊文件或无法安全计量的临时树会立即失败闭合。
- Unix 使用独立进程组和父进程存活监视，监视进程只保留自己的父存活句柄，避免并发命令因继承其他输出管道而延迟回收；Windows 使用先挂起、加入启用关闭即终止策略的 Job Object 后再恢复执行。即使解析进程被强制终止，监视者或操作系统也会清理 Git 后代。故障注入覆盖永不退出、调用前和运行中取消、调用前超时、父进程强制退出、并发描述符隔离、双输出流精确边界与溢出、磁盘精确边界与增长、磁盘根替换、后代回收和临时目录清理。
- 清单格式 2、锁文件格式 2、Git 精确修订和缓存 generation 格式均未改变。超过新硬预算的旧对象库在离线模式下会明确拒绝；联网重试会保留旧现场并建立独立修复对象库，无须改写源码或锁文件。

### ABI v2 原生装载边界

- 磁盘原生模块只允许当前模块所属的锁定包加载自身制品或从它的直接锁定依赖边选择，并从只读依赖解析阶段已经打开的 generation 文件令牌读取；别名边仍按真实包名匹配，同名路径、整个包根替换和调用方越权点名传递依赖都不能改变选择或实际装载字节，运行时解析也不会改写锁文件。
- YXB 保持从归档内已验证字节装载；磁盘与归档路径统一复核 ABI、目标、大小、SHA-256、宿主二进制格式和动态依赖元数据。
- ABI v2 三平台只接受当前架构的 64 位宿主动态库。Linux 拒绝 RPATH/RUNPATH、路径或动态令牌型 `DT_NEEDED`和额外装载标签；macOS 只允许固定系统库路径并拒绝未覆盖的装载命令；Windows 拒绝延迟导入。
- ABI v1/v2 的共享打开器在 Unix 立即解析全部符号，在 Windows 只搜索私有暂存目录和 System32，不再搜索应用目录、当前目录或用户追加目录。
- ABI v1/v2 公开布局、清单格式 2、锁文件格式 2 和 YXB 格式 1 均未改变；依赖未锁定相邻动态库的 ABI v2 制品，以及依赖应用目录或用户 DLL 目录的 Windows ABI v1 制品，须改为静态链接或系统依赖后重新构建。

### 兼容与安装

- 继续使用 1.1.x 补丁版本；语言语法、标准库签名、清单格式 2、锁文件格式 2、YXB 格式 1、字节码格式 2、原生 ABI v1/v2 和工程协议 1 均未提升版本。
- 首次升级会按锁定摘要重建新的依赖缓存；旧缓存保持可恢复，不要求改写源码或锁文件。依赖非系统相邻动态库、超大单模块或超过新硬预算的项目须按迁移说明调整。

## 1.1.16

### 二进制值与 HTTP 响应预算

- 二进制值的宿主硬上限统一为 16 MiB；HTTP 默认累计响应预算保持 4 MiB，实际预算取应用请求值与宿主上限的较小值。
- 套接字单次读取上限独立保持 4 MiB，不再与 HTTP 累计响应预算混用；超过 4 MiB 且不超过 16 MiB 的 HTTP 二进制响应可完整返回。
- 已知长度和流式正文都在读取上限外再探测一个字节，精确边界成功、超限以稳定`NET_LIMIT`失败，不返回截断成功值。
- 树解释器、字节码 VM、YXB 与嵌入运行时使用同一预算语义；嵌入宿主只能收紧应用请求，不能突破 16 MiB 硬上限。

### 旧字节码制品诊断

- 字节码格式 1 缺少当前运行时要求的完整模块与类型身份，不能安全自动转换；当前运行时继续严格拒绝，不猜测恢复类型链接。
- 裸字节码与 YXB 分别使用`BYTECODE_FORMAT_UNSUPPORTED`和`YXB_BYTECODE_UNSUPPORTED`，报告检测格式、支持格式、迁移结论与完整重建命令。
- 加入由言序 1.1.12 构建的真实旧 YXB fixture，覆盖旧源码、旧制品、当前制品与损坏归档回归。

### 兼容与工程门禁

- 新增以`v1.1.15`为基线的结构化 SemVer 门禁，覆盖语法、Tree-sitter、标准库、公共格式、ABI、工程协议、稳定 CLI、嵌入接口和全部 JSON Schema。
- `docs/project/status.md`成为当前版本、格式与兼容计数的唯一状态真源，历史台账移入归档并由测试核对实际实现。
- 不改变语言规范版本、标准库签名、清单格式 2、锁文件格式 2、YXB 格式 1、字节码格式 2、原生 ABI v1/v2 或工程协议 1；Rust 嵌入公开接口保持兼容。
- 依赖超过 16 MiB 单个二进制值或 HTTP 正文的应用须改为流式或分块处理；其他 1.x 源码、当前 YXB 与锁文件无需迁移。

## 1.1.3

### 安全

- 持久`Engine`在运行失败后强制重建运行时并清空类型历史，避免部分状态与静态历史分叉；新增显式`Engine::reset`。
- Release 标题使用已校验版本值，制品全部上传前保持草稿；所有 GitHub Actions 固定到完整提交 SHA。
