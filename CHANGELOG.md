# 变更记录

## 1.1.14

### HTTP 二进制响应预算

- `网络.请求字节`的最大响应字节参数改用 HTTP 响应预算校验，不再错误套用套接字 4 MiB 单次接收上限。
- 树解释器与 VM 保持一致；超过 4 MiB 的合法预算现在交由 HTTP 传输按累计响应大小执行，并继续以`NET_LIMIT`报告实际超限。
- `套接字.接收`、`套接字.接收字节`、`套接字.精确读取`和 UDP 接收函数仍保留 4 MiB 单次接收硬上限。

### 兼容与升级

- 不改变语言规范 1、标准库签名、原生 ABI v1/v2、清单、锁文件、YXB 或 Bundle 格式。
- 使用超过 4 MiB 二进制 HTTP 响应预算的应用应升级运行时并重新构建 Bundle；既有源码与锁文件无需迁移。

### 安装

macOS 与 Linux 可固定安装本版本：

```sh
curl -fsSL https://get.yanxu.dev | YANXU_VERSION=1.1.14 sh
```

Windows PowerShell 可先设置`$env:YANXU_VERSION = "1.1.14"`，再运行
`irm https://get.yanxu.dev/windows | iex`。

## 1.1.13

### 模块限定类型与跨模块对象

- 类型路径改为结构化段，`模块.类型`可用于注解、联合、可空、泛型、法类型、`承`、`纳`和`是`；同模块裸类型保持兼容。
- 类与协议使用包含规范模块所有权的`TypeId`，不同模块同名类型互不碰撞，同一模块的不同导入别名保持同一身份。
- 新增顶层`公 引「模块」为 别名；`，支持任意深度 facade 重导出；检查器、树解释器和 VM 同步支持跨模块继承、协议、动态派发、`父.方法`、静态成员和私有边界。
- 字节码块升级为格式 2，保存`ModuleId`、`TypeId`及结构化运行时类型链接；YXB 外层格式仍为 1，但声明`bytecode_format: 2`。
- 规范模块身份校验以平台无关规则拒绝 Unix、Windows 盘符和 UNC 绝对路径，避免同一归档身份在不同宿主上得到不一致结果。
- `yanxu 文 --json`升级为模块 API 清单 v2，新增类型所有者、完整限定名称、结构化类型引用与重导出暴露路径；Markdown 与 LSP 可跟随限定路径和 reexport 到原声明。

### 兼容与迁移

- 有意移除“检查过导入模块后可偶然裸写其类型”的未规定行为；改写为`导入别名.类型`。当前模块裸类型、普通模块值访问和现有对象语法不变。
- 字节码格式 1 不再由当前运行时读取，旧 YXB 需要从源码重新构建；标准库`stdlib-api-v1`不受模块 API 清单版本变化影响。
- 迁移步骤见[模块限定类型迁移](reference/migration-module-qualified-types.md)，完整示例见[`examples/module_oop`](examples/module_oop)。


Windows PowerShell 可先设置`$env:YANXU_VERSION = "1.1.13"`，再运行

## 1.1.9

### 执行预算

- 新增全局旗标`--max-steps <N>`和环境变量`YANXU_MAX_STEPS`，可为源码执行、VM、包运行和应用归档统一设置单次执行步数预算；命令行旗标优先于环境变量。
- 非法或为零的预算值现在会在执行前以明确诊断拒绝；步数超限诊断同时提示两种配置入口。

### 常驻宿主回调

- GUI、计时器和原生扩展等常驻程序的宿主事件回调改为独立计量步数，避免多个正常回调的累计执行量误触全程预算。
- 单个回调内的死循环仍会耗尽预算并中止；回调完成或失败后恢复外层执行计量，不会放宽外层余额。

### 兼容与升级

- 不改变语言规范 1、标准库 API、原生 ABI v1/v2、清单、锁文件、YXB 或 Bundle 格式；未配置预算时继续使用默认值 1000000。
- GUI 应用建议升级到言序 1.1.9 后重新构建，以获得按事件隔离的回调预算。

## 1.1.8

### 新功能

- 把 Windows VM 所有者线程的 8 MiB 栈保留量纳入可检查的运行时构建契约；本补丁不新增语言语法、标准库或原生 ABI 接口。

### 修复内容

- 修复 Windows 可执行文件使用链接器默认 1 MiB 主线程栈时，VM 所有者线程在原生事件循环回调中执行正常言序方法嵌套可能触发栈溢出的问题。
- x86_64 与 ARM64 MSVC 目标的`yanxu.exe`统一保留 8 MiB 栈；桌面 Bundle 携带同一运行时，因此无需应用或 GUI 包使用线程绕行或裁剪控件树。
- 新增 Windows PE32+ 回归，直接校验发行运行时的`SizeOfStackReserve`，防止链接参数或构建流程退化。

### 兼容与迁移

- 不改变语言规范 1、标准库 API、原生 ABI v1/v2、清单/锁文件、YXB 或 Bundle 格式；命令行、嵌入和非 Windows 目标保持兼容。
- 新建 GUI 清单及官方 GUI 示例的最低运行时更新为 1.1.8；已有 GUI 项目只需升级运行时并重新构建 Windows Bundle，不必修改源码或原生包制品。

### 安装与升级

macOS 与 Linux 可固定安装本版本：

```sh
curl -fsSL https://get.yanxu.dev | YANXU_VERSION=1.1.8 sh
```

Windows PowerShell 可先设置`$env:YANXU_VERSION = "1.1.8"`，再运行
`irm https://get.yanxu.dev/windows | iex`。GUI 项目把最低运行时改为`>=1.1.8`、重新锁定，
并在对应 Windows 原生目标重新构建 Bundle。

### 平台、制品与校验

- 正式支持 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标；每个目标提供运行时归档和独立 SHA-256 文件。
- 发布流程在六种原生 runner 构建、运行并通过安装脚本复验；两个 Windows Release 二进制还会直接核对 PE32+ 栈保留量。

### 第三方依赖与已知限制

- 本补丁不新增或升级第三方运行时依赖，继续沿用 1.1.7 已审计的许可证与安全边界。
- 修复覆盖官方`yanxu.exe`及由它生成的 GUI Bundle；把言序嵌入自定义宿主线程的应用仍须由宿主设置足够的线程栈。由 1.1.7 打包的旧 Windows Bundle 必须重新构建，不能只替换 YXB。

## 1.1.7

### 原生 ABI v2 与宿主事件循环

- 保留 ABI v1 独立入口并新增 ABI v2：递归类型值、持久回调、宿主函数表、父子资源、代际句柄和精确单次析构。
- 原生线程只向有界、可合并队列投递深拷贝事件；回调在 VM 所有者线程执行，重入、过期句柄、队列压力、panic 和异常输出均有回归。
- Windows 动态库只从内容寻址私有目录及可信系统位置解析依赖；三平台继续在校验目标、大小与 SHA-256 后装载只读副本。

### 言窗与桌面 Bundle

- 清单格式 2 新增图形界面、剪贴板、文件对话框、通知、托盘、外部地址和全局快捷键权限，并把应用名称、标识、版本、图标与窗口约束写入 YXB。
- YXB 可携带多平台锁定原生元数据和当前目标真实制品；运行不再依赖源码、包缓存或联网下载。
- 新增 `yanxu compile <项目> --release --bundle`：生成 macOS `.app`、Windows GUI 应用目录或 Linux AppDir，含图标、平台元数据、YXB、原生库、资源、许可证、签名计划和逐文件摘要。
- Bundle 使用同级私有暂存、完整校验和原子替换；启动器在读取内嵌 YXB 前验证外层清单、入口及全部索引文件，源码删除运行、路径越界、架构错配、清单链接和摘要篡改均有自动化测试。

### 工具链

- 工程协议新增 GUI 模板与 Bundle 操作；LSP 可离线补全锁定包公开成员，`yanxu 文 --json`生成机器可读包 API。
- 官方言窗、言包 0.4.0 和 VS Code 扩展 1.4.0 共同覆盖 GUI 创建、运行、补全、构建与六目标 CI；不在本版本开发过程中创建标签或公开 Release。

## 1.1.6

### 1.1.5 遗留修复

- 正式纳入嵌套在路径依赖内部的应用依赖边解析、包管理/CI 回归、显式`任意`静态类型、YXB 确定性压缩与调试源码压紧、YXB 解压大小限制及最新 Tree-sitter 生成制品。
- 保持旧版未压缩 YXB v1 可读；新制品继续使用确定性压缩，不改变语言规范 1 的源码语法。

### YXB 安全与资源预算

- YXB 执行权限改为“宿主授权上限与应用申请权限的交集”，应用不能提升文件、网络外连、TCP 监听、UDP 绑定、环境、进程或原生扩展权限；公共`PermissionSet::intersect`供嵌入宿主复用同一语义。
- 读取前检查普通文件与 64 MiB 压缩体积，解压 JSON 限制为 256 MiB；限制模块、单模块、指令、函数、类、调试位置、资源条目、单资源和资源总量。
- 新增模块 ID/入口/内部导入、字节码版本、资源安全相对路径/索引/摘要和全层重复 JSON 键验证；覆盖重新计算摘要后的权限篡改、压缩炸弹、截断/畸形流、超大文件与超多条目。
- YXB 写入运行时版本、构建提交、目标和构建模式。文档明确普通 SHA-256 只校验完整性，不证明发布者身份；可选数字签名留给后续格式提案。

### 原生扩展与 ABI

- 动态库从已打开的普通文件限长读取并校验目标/SHA-256，再复制到进程私有、内容寻址的只读路径，复核摘要后装载；原始路径随后被替换不会改变本次装载代码。
- 原生 JSON 和资源输出改用 RAII 所有权守卫；调用失败、空指针、异常长度、JSON 解析失败、未注册资源类型和未知输出种类进入统一释放路径。
- 限制动态库、模块/函数/资源名称、错误码和错误消息长度；增加重复描述符、异常输出、资源单次析构和回调重入/递归测试。
- 明确原生动态库属于进程内受信任代码，不受言序语言沙箱完全约束，并记录 Windows、Linux、macOS 的暂存、只读、卸载和系统加载策略。

### 包管理与可追溯构建

- 清单、锁文件和 vendor 清单改为同目录临时写入、同步与原子替换；显式更新不再先删除旧锁，解析失败保留上一份完整锁文件。
- 同一项目的依赖编辑、安装、更新、YXP 打包和 YXB 构建在完整输入读取周期使用跨平台项目锁；新增 8 路并发编辑/构建、完全离线不联网、损坏 registry 缓存恢复和失败回滚回归。
- `yanxu version --json`模式 2 新增完整构建提交 SHA、Cargo 目标和 debug/release 模式；工程协议与 YXB 复用相同构建身份。
- CLI 的包/YXB/standalone 命令、VM 原生调用表、包原子存储/项目锁和安全归档解压建立独立模块边界；安全归档定位不再依赖隐式`expect`。

### CI 与发布

- 三平台 Rust、WASI、Rust/C/WASI 嵌入、Tree-sitter、安装器、YXB 安全、原生 ABI 异常及包并发/离线验收统一为可复用质量工作流，并上传版本、兼容、标准库和总体验收 JSON。
- 修改 Cargo 版本不再自动抢先创建标签；核心 Release 必须精确匹配`origin/main`验收提交，并在完整门禁通过、六平台二进制和 SHA-256 全部上传后公开，再在六种原生目标上通过安装脚本复验指定版本；失败会撤销稳定最新版资格。
- 安装器只查询 GitHub 的最新稳定 Release，支持固定版本与受校验的本地制品目录；损坏摘要、不可运行候选和标签/二进制版本不一致均不会覆盖旧安装。
- `yanxu-package`使用独立标签、独立质量门禁和独立 Release，只发布共享包 crate 与校验文件，不混入核心运行时 Release。

## 1.1.5

### 工程化与包生态

- 拆出独立`yanxu-package`核心库；清单格式 2 新增最低言序版本、开发依赖、别名、显式导出、资源、工作区、构建与原生制品声明，并兼容读取格式 1。
- 锁文件格式 2 在构建前固定完整直接/传递依赖图、别名边、精确版本、来源、Git 提交、内容校验和、目标与原生制品；运行时不再临时解析传递依赖。
- `包:别名/导出`支持包子模块和显式导出表，顶层源码不能绕过清单导入未声明的传递依赖。
- 新增版本化 JSON 工程协议，覆盖握手、清单创建/编辑、解析、完整图、更新预演、依赖解释、工作区、诊断、确定性打包、离线辖制与审计。
- 确定性 YXP 打包拒绝链接、特殊文件、越界路径与资源爆炸；`vendor`可按锁文件恢复脱离原来源位置的完整图。

### 编译与应用制品

- 新增`yanxu compile`、`yanxu run`和 YXB 应用归档；制品包含预编译入口/项目/依赖模块、资源、权限、调试路径与内容校验和，运行不读取或解析原始`.yx`。
- YXB v1 改用确定性压缩载荷并压紧重复调试源码；仍兼容读取早期未压缩 v1，同时限制解压后大小。言镜验收样本由约 250 MiB、百秒级构建降至约 305 KiB、4 秒。
- `--standalone`把言序 VM 与 YXB 封装为 Windows、Linux 和 macOS 自包含程序；目标机无需安装言序。
- 输入未变时构建缓存直接复用制品；相同输入多次构建产生逐字节一致的 YXB/YXP。
- 新增`标准:资源`与`标准:进程`；两者均同步树解释器、VM、静态类型和机器 API 清单，标准模块数增至 25。

### 原生扩展与工具链

- 新增稳定原生扩展 ABI v1、C 头文件和 Rust 示例，支持 JSON 函数、常量、不透明宿主资源、结构化错误、释放器与言序回调。
- 加载器强制校验目标平台、SHA-256、ABI/结构版本、符号、描述符和`原生扩展`权限；WASI 明确禁止动态库加载。
- LSP 新增包别名/公开子模块补全、锁定版本/来源信息、未声明依赖和清单-锁漂移诊断。
- 修复静态检查器把显式`任意`误当成普通具名类型的问题；`任意`现在可接纳并传播空、标量、容器和用户对象等动态值。
- 持久嵌入引擎选择树/VM 后端时改用间接持有，避免应用模块缓存扩展引起宿主引擎栈对象膨胀。

## 1.1.4-A

### 包管理

- Git 依赖的符号修订在显式更新时会重新从远端获取，不再复用缓存中的陈旧本地分支；锁文件仍只记录解析后的精确提交。
- 包模块加载自动信任已经通过清单、版本、锁文件和内容校验的源码根，不再要求应用把全局 Git 缓存写入`[权限].文件`。
- 顶层清单显式锁定的同名依赖优先；顶层未声明时回退到当前依赖包自己的清单，兼顾统一版本选择与传递依赖。
- 上述包源码授权只用于模块加载，不合并依赖包的文件、网络、监听、环境或进程权限。

## 1.1.4

### 新增

- 新增不可变`字节串`类型与`标准:字节`，覆盖 UTF-8 编解码、实际字节长度、半开切片、拼接、查找和 0..255 数列互转；单值上限为 4 MiB。
- `文件`新增二进制读取、写入、追加与含种类/字节数/只读/修改时间的状态查询。

- `网络.请求字节`支持自定义请求首部、二进制请求正文和二进制响应，并拒绝 CRLF、非法首部及调用方控制的逐跳/长度首部。
- 新增操作系统安全随机、HMAC-SHA256、常量时间比较、HTTP 日期格式/解析和`环境.参数`；CLI 使用`--`分隔程序参数。

- 嵌入`Execution`与 C ABI 可返回原始字节数列；新增二进制文件、HTTP、TCP EOF/短读、密码向量和双后端一致性回归。
- 本版明确不包含 HTML、HTTP 服务端、Web 框架、真正异步 I/O、服务端 TLS、HTTP/2/3、WebSocket、原生编译器或自举。

## 1.1.3

### 安全

- HTTP 重定向改为逐跳重新校验目标主机，DNS 解析后再校验实际 IP；回环、私网、链路本地与特殊地址须精确授权，防止重定向越权与 DNS rebinding/SSRF。
- 拆分`network`外连、`tcp_listen`监听和`udp_bind`绑定权限；沙箱默认不允许任何监听，包清单新增`TCP监听/UDP绑定`声明。
- 远程包制品改为进程内安全解压：拒绝符号链接、硬链接、设备文件、管道、越界/过长路径和重复写入，并限制压缩体积、单文件、展开总量与条目数。

### 修复

- HTTP、TCP 连接与 UDP 发送的 DNS、多地址尝试和传输共享同一调用级截止时间，不再在 DNS 结束后重置超时。
- TCP 接收保留未完成 UTF-8 字符的内部字节，正确组合跨多次底层读取的中文；每个运行时最多同时持有 128 个套接字资源和 16 个 TCP 监听器。
- 持久`Engine`在运行失败后强制重建运行时并清空类型历史，避免部分状态与静态历史分叉；新增显式`Engine::reset`。
- Release 标题使用已校验版本值，制品全部上传前保持草稿；所有 GitHub Actions 固定到完整提交 SHA。
